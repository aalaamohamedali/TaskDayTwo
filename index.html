2. What is a Pull Request and Request? Difference between them?

  • Pull Request (PR): A pull request is a GitHub feature used to notify others that you want to merge changes (from your branch or fork) into another branch (usually main) in a repository.
  • Request (in general): A request is just a generic term meaning you ask for something. A pull request is a type of request.

4. What is the command to delete the branch "new-email"?

  • Locally 
                git branch -d new-email  # Use -D to force delete
  • Remotely 
                git push origin --delete new-email

5. What is the command to push the current repository to the remote origin?

        git push origin HEAD
OR: if you are on the main branch: 
        git push origin main

6. What is Git Rebase and how to use it?

  • Git Rebase is used to reapply commits on top of another base tip. It helps to maintain a clean and linear history. 
                # Rebase your current branch onto main
            git checkout feature-branch
            git rebase main

        
                # Resolve conflicts manually, then:
            git add .
            git rebase --continue

7. What is the command to get all the change history of the remote repository "origin"?

git fetch origin
        git log origin/main --oneline
Or to see all branches: 
git log --all --oneline

8. What is the command to show the differences between the current branch and the branch "new-email"?

        git diff new-email
OR: 
        git diff HEAD..new-email

9. What are Tags? Make five tags and release two of them

    Tags mark specific commits, usually for releases (e.g., v1.0.0).
  • Create 5 tags: 
                git tag v1.0.0
            git tag v1.1.0
            git tag v2.0.0
            git tag beta
            git tag stable
  • Push all tags: 
                git push origin --tags
